# index-portal-journal

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何单个域名或具体网站，而是作为通用索引、信息展示或功能入口的集合。

## 项目简介

`index-portal-journal` 是一个静态页面归档仓库。每个 HTML 文件均为独立、自包含的页面，可用于记录、展示或跳转至各类资源。仓库结构清晰，便于快速浏览和引用。

## 目录说明

```
index-portal-journal/
├── pages/          # 存放所有独立的 HTML 页面文件
├── assets/         # 公共资源（CSS、图片、字体等，如有需要）
├── index.html      # 仓库入口页面（可选，用于索引所有页面）
└── README.md       # 本文件
```

- `pages/`：每个文件为一个独立页面，文件名建议简洁且能反映内容主题。
- `assets/`：共享资源文件，按需组织。
- `index.html`：可选的根目录入口，用于汇总展示所有页面的链接。

## 页面归档说明

- 所有页面均为纯 HTML 文件，不依赖后端服务。
- 每个页面应包含基本的 HTML 结构（`<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` 等）。
- 页面之间无强关联，可单独部署或引用。
- 建议在页面内添加简单的导航或返回链接，以提升浏览体验。

## 维护说明

- 新增页面：请在 `pages/` 目录下创建新的 `.html` 文件，并更新 `index.html`（如果存在）中的链接。
- 修改页面：直接编辑对应文件，保持原有结构。
- 删除页面：移除文件并清理相关链接。
- 资源管理：公共资源放入 `assets/`，避免在页面内硬编码路径。
- 提交信息：建议使用简洁的描述说明变更内容，如“添加页面：xxx”或“更新页面：xxx”。

## 注意事项

- 本仓库仅用于归档和展示，不承担任何外部链接或内容的审核责任。
- 请勿在页面内包含违法、侵权或恶意内容。
- 建议定期检查页面链接有效性，确保内容可正常访问。
